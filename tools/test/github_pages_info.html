🌐 GitHub Pages 環境說明

📍 當前環境

平台: GitHub Pages

URL: https://kelvinhuang0327.github.io

類型: 靜態網站托管

限制: 不支持服務器端代碼

⚠️ 重要限制

GitHub Pages 不支持 Node.js 後端

GitHub Pages 是一個靜態網站托管服務，只能托管 HTML、CSS、JavaScript 等前端文件，無法運行 Node.js 後端服務器。

這意味著:

  • ❌ 無法使用 Token 儲存功能
  • ❌ 無法使用自動備份功能
  • ❌ 無法使用數據同步功能
  • ❌ 無法使用跨瀏覽器數據共享

📊 功能對比

✅ 可用功能

  • 數據查看和統計
  • 圖表和視覺化
  • 日曆視圖
  • CSV/Excel 匯入
  • 數據匯出
  • 本地數據管理

❌ 不可用功能

  • Token 儲存管理
  • 自動備份到 GitHub
  • 數據同步
  • 跨瀏覽器共享
  • 數據健康檢查
  • 快速修復功能

🚀 解決方案

方案 1: 使用本地環境 (推薦)

在您的電腦上運行完整的應用，包括後端服務：

  1. 確保已安裝 Node.js
  2. 在項目目錄運行: node server.js
  3. 訪問: http://localhost:8000
🚀 切換到本地環境

方案 2: 部署到支持後端的平台

將應用部署到支持 Node.js 的雲端平台：

  • Vercel: vercel.com (推薦)
  • Railway: railway.app
  • Render: render.com
  • Heroku: heroku.com

💡 快速開始本地環境

如果您想使用完整功能：

  1. 下載項目到本地
  2. 安裝 Node.js (如果還沒有)
  3. 在終端中運行: node server.js
  4. 打開瀏覽器訪問: http://localhost:8000

這樣您就可以使用所有功能，包括 Token 儲存和自動備份！

📞 需要幫助？

如果您需要幫助設置本地環境：

  • 查看項目的 README.md 文件
  • 使用 start_services.sh 腳本
  • 參考 DEPLOYMENT_GUIDE.md 部署指南
